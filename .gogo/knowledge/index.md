# .gogo/knowledge — what the gogo pipeline reads

**Purpose:** the index / purpose-map for this folder. These files are the
project-specific configuration the gogo pipeline (plan → implement → review →
test → report) reads to stay grounded in *this* project.

Generated by `/gogo:build`. Re-run anytime — it preserves your edits (see below).

## Proxy convention
Each file is one of two modes (its `gogo:meta` header says which):
- **proxy** — `Mode: proxy` + `Source:` link(s) to the real doc. Body is a short
  gogo-specific summary; the linked file is the source of truth. Don't duplicate.
- **owned** — `Mode: owned`. No upstream doc existed, so gogo authored this file
  and may edit it freely.

Anything under `## gogo overrides`, and any `Mode: owned` file, is preserved
across `/gogo:build` re-runs.

Command pipeline (intended design)

flowchart TD
  PLAN[/"plan.md (accepted)"/]:::art --> IMP
  CR[/"coding-rules"/]:::art --> IMP
  ISS -. "--issues" .-> IMP
  IMP["gogo:implement (validate in/out)"]:::cmd
  IMP --> CODE["code"]:::out
  IMP --> CHARTS[/"charts: flow · sequence · class · activity"/]:::art
  CODE --> REV
  CHARTS --> REV
  CRS[/"code-review-standards"/]:::art --> REV
  PLAN --> REV
  REV["gogo:review (validate in/out)"]:::cmd
  REV --> ISS[/"issues.json (living, typed)"/]:::art
  ISS --> G1{"open issues?"}:::gate
  G1 -->|"yes - fix"| IMP
  G1 -->|"clean"| TEST
  TS[/"test-strategy"/]:::art --> TEST
  PLAN --> TEST
  CHARTS --> TEST
  TEST["gogo:test (validate in/out)"]:::cmd
  TEST --> TISS[/"test issues.json"/]:::art
  TISS --> G2{"open issues?"}:::gate
  G2 -->|"yes - fix"| IMP
  G2 -->|"clean"| REP["gogo:report (report.md + knowledge)"]:::cmd
  REP --> DONE([done]):::io

Issue lifecycle (the living issues.json)

stateDiagram-v2
  [*] --> open: review / test finds it
  open --> fixed: implement --issues (sets fix_summary)
  fixed --> verified: re-review confirms
  fixed --> open: re-review still fails
  open --> wontfix: user decision
  open --> new: surfaced this round
  verified --> [*]
  wontfix --> [*]

Validated hand-off (review → implement)

sequenceDiagram
  actor U as user / go
  participant R as gogo:review
  participant C as gogo-contracts
  participant I as gogo:implement
  U->>R: review(slug)
  R->>C: validate-in(plan, standards, charts)
  C-->>R: ok
  R->>R: review the diff
  R->>C: validate-out(issues.json)
  C-->>R: schema ok (ids unique, enums valid)
  R-->>U: issues.json (3 open)
  U->>I: implement --issues issues.json
  I->>C: validate-in(plan, coding-rules, issues.json)
  C-->>I: ok
  I->>I: fix open issues
  I->>C: validate-out(issues.json + charts)
  C-->>I: ok (status=fixed, fix_summary set)
  I-->>U: updated issues.json + as-built charts

Data model — Project · Sources · Plans · WorkItems (correlation[] in state.md)

classDiagram
  direction TB

  class Project {
    +string name
    +string description
    ~/.gogo/projects/<name>/config.json
    .knowledge/ (project knowledge)
    .gogo/plans/ (project-scoped plans)
  }
  class Source {
    +string path  (a repo/service with .gogo/)
    +string name
    +string mainBranch
    +int concurrentWorkItems
    +string color
  }
  class Plan {
    +string id  "plan-<hash> = the correlation id"
    +string title
    +string description
    +string status  "draft | active"
    ~/.gogo/projects/<name>/.gogo/plans/<id>.md
  }
  class Target {
    +string source  "which source this plan targets"
    +string slugHint
  }
  class WorkItem {
    +string slug
    <source>/.gogo/work/feature-<slug>/
    CLI-READ-ONLY (skills write it)
  }
  class StateMd {
    +string phase
    +string status
    +correlation[] plan-XXXX  "ADDITIVE optional LIST (L1)"
  }

  Project "1" *-- "many" Source : links (config.json)
  Project "1" *-- "many" Plan : owns (.gogo/plans/)
  Plan "1" o-- "1..*" Target : targets sources
  Target ..> Source : resolves to
  Plan "1" ..> "0..*" WorkItem : spawns via claude -p (gogo:plan)
  WorkItem "1" *-- "1" StateMd : carries
  StateMd "correlation[]" ..> "many" Plan : belongs to (many-to-many)

  note for StateMd "Board reads correlation[] directly from state.md\n(no CLI store overlay) -> ⛓ plan-XXXX chips + #plan-XXXX filter"
  note for WorkItem "Spawn = launch claude -p; the SKILL writes .gogo/work/\nand stamps correlation. The CLI NEVER writes a source's .gogo/work/."

Runtime — plan -> spawn (claude -p gogo:plan) -> correlation stamped in state.md

sequenceDiagram
  autonumber
  actor User
  participant TUI as gogo TUI (plans tab / detail)
  participant Plans as plans store (~/.gogo/projects/<n>/.gogo/plans/)
  participant Launch as launch.PlanIntent
  participant Claude as claude -p (gogo:plan SKILL)
  participant Work as source .gogo/work/feature-<slug>/
  participant Board as board reader (contract.LoadProject)

  User->>TUI: n new plan / A author with claude
  TUI->>Plans: Create Plan{id: plan-XXXX, targets, status:draft}
  Note over Plans: plan-XXXX = the correlation id, minted at creation

  User->>TUI: open plan detail, press c "create work item" on source S
  TUI->>Launch: PlanIntent(root=S, body, correlation="plan-XXXX")
  Launch-->>Claude: claude -p "/gogo:plan <body> --correlation plan-XXXX"  (anchored at S, single argv element)
  Note over TUI,Claude: CLI writes NOTHING under S/.gogo/work/ (hard invariant)

  Claude->>Work: write feature-<slug>/ (derives the final slug)
  Claude->>Work: stamp state.md → - **correlation:** [plan-XXXX]
  Claude-->>Plans: (plan goes active; member {S, slug} recorded)

  Board->>Work: read state.md (LLM-free, ms-fast)
  Work-->>Board: Feature.Correlations = [plan-XXXX]
  Board-->>User: card shows ⛓ plan-XXXX · #plan-XXXX filters members across sources

  User->>TUI: add an EXISTING work item to plan-YYYY (retroactive, FR16)
  TUI-->>Claude: re-stamp → state.md correlation list gains plan-YYYY (many-to-many)

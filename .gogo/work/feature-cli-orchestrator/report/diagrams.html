The Go orchestrator loop

flowchart TD
  run["gogo run <slug>"] --> reg{"session registry
.gogo/resources/cli/sessions/<slug>.json"}
  reg -->|"dev uuid known"| resumeDev["claude -p --resume <dev-uuid>
/gogo:implement <slug> --in-session"]
  reg -->|"first run"| newDev["claude -p --session-id <dev-uuid>
/gogo:implement <slug> --in-session"]
  newDev --> devOut["implement/result.json + charts/
state.md: implementing"]
  resumeDev --> devOut
  devOut --> review["claude -p --session-id <fresh-uuid>
/gogo:review <slug> (fresh eyes)"]
  review --> revOut["review/issues.json + result.json"]
  revOut --> route1{"contract.Route(issues, result)"}
  route1 -->|"open agent-fixable"| resumeDev
  route1 -->|"needs-user-decision"| gate["pause queue → tmux attach gate
(existing launch.AttachArgs)"]
  route1 -->|"clean"| test["claude -p --session-id <fresh-uuid>
/gogo:test <slug> (fresh eyes)"]
  test --> testOut["test/issues.json + result.json"]
  testOut --> route2{"contract.Route(issues, result)"}
  route2 -->|"open agent-fixable"| resumeDev
  route2 -->|"needs-user-decision / e2e-blocked"| gate
  route2 -->|"all green"| report["claude -p /gogo:report <slug>
(fresh one-shot)"]
  report --> uat["state.md: awaiting-uat
(UAT gate — the human's)"]
  gate -.->|"human answers, state.md resumable"| run
  route1 -.->|"same finding > 3 rounds
or cost ceiling hit"| gate

Two orchestrators over one shared core

flowchart TD
  subgraph drivers["Two orchestrators (interchangeable drivers)"]
    inchat["in-chat orchestrator
skills/gogo (default, no deps)"]
    cli["Go CLI orchestrator
cli/internal/orchestrator (opt-in: tmux + claude)"]
  end

  subgraph core["ONE shared core (never duplicated)"]
    cmds["phase commands + skills
/gogo:implement · /gogo:review · /gogo:test · /gogo:report"]
    contracts["typed contracts
state.md · */issues.json · */result.json · events.jsonl"]
    route["the ONE routing rule
review/test '④ Route': open/new count + needs-user-decision"]
  end

  inchat -->|"Task subagents,
orchestrator freezes while they run"| cmds
  cli -->|"claude -p sessions,
Go process never freezes"| cmds
  cmds --> contracts
  inchat -->|"applies (prose)"| route
  cli -->|"applies (contract.Route, cites the skill)"| route
  route --> contracts

  cli -.->|"warm dev via --resume,
fresh review/test via new --session-id"| sessions["phase-session registry
.gogo/resources/cli/sessions/"]

Warm dev resume across fix rounds

sequenceDiagram
  actor U as User
  participant G as gogo run (Go loop)
  participant R as registry (.gogo/resources/cli/sessions)
  participant D as dev session (claude -p, one uuid)
  participant V as review session (fresh uuid/round)
  participant T as test session (fresh uuid)

  U->>G: gogo run 
  G->>R: read/create dev uuid
  G->>D: claude -p --session-id DEV "/gogo:implement  --in-session"
  D-->>G: exit · implement/result.json + charts/ (warm context retained by claude)
  G->>V: claude -p --session-id REV1 "/gogo:review "
  V-->>G: exit · review/issues.json (2 open)
  G->>D: claude -p --resume DEV "/gogo:implement  --issues review/issues.json --in-session"
  Note over D: NO codebase re-read — resumes warm
  D-->>G: exit · issues.json fixed back
  G->>V: claude -p --session-id REV2 "/gogo:review " (fresh eyes again)
  V-->>G: exit · review clean
  G->>T: claude -p --session-id TEST1 "/gogo:test "
  T-->>G: exit · all green
  G->>R: record costs/turns per session (--output-format json)
  G-->>U: report written · state.md awaiting-uat (run /gogo:done to ship)
